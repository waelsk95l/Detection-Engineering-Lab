# Case 10 – Elastic Automation: Suspicious PowerShell Detection

## Overview

This project demonstrates automated detection of suspicious PowerShell activity using:

- Elasticsearch
- Kibana
- Sysmon
- Winlogbeat
- Python Automation

The lab simulates suspicious PowerShell behavior and automatically detects malicious command-line activity using Python automation and Elastic queries.

---

# Objectives

- Detect suspicious PowerShell execution
- Detect encoded PowerShell commands
- Detect DownloadString and IEX usage
- Automate Elastic threat hunting with Python
- Practice SOC Detection Engineering workflows

# Detection Logic

The detection searches for suspicious PowerShell activity containing:

- `-enc`
- `EncodedCommand`
- `IEX`
- `DownloadString`
- `-nop`
- `ExecutionPolicy Bypass`

---

# Elastic Queries

## Basic PowerShell Query

```kql
process.name:"powershell.exe"
```

## DownloadString Detection

```kql
process.command_line:*DownloadString*
```

## IEX Detection

```kql
process.command_line:*IEX*
```

## Encoded Command Detection

```kql
process.command_line:*enc*
```

---

# Python Automation

Main automation script:

```text
elastic_powershell_detector.py
```

The script connects to Elasticsearch and automatically identifies suspicious PowerShell command-line activity.

# SOC Analyst Notes

PowerShell is heavily abused by attackers for:

- Fileless malware
- Remote payload download
- Command execution
- Obfuscation
- Living-off-the-land attacks

Even failed execution attempts may still indicate malicious intent and should be investigated.
